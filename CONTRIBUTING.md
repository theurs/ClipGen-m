# Contributing to ClipGen-m

Thank you for your interest in contributing to ClipGen-m! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/clipgen-m.git`
3. Create a branch for your changes: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes: `git commit -m "Add your descriptive commit message"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Create a Pull Request

## Development Setup

1. Install Go 1.25 or later
2. Clone the repository
3. Run `go mod download` to install dependencies
4. Build with `go build` or use the provided `build.bat` scripts

## Code Style

- Follow Go standard formatting with `gofmt`
- Use descriptive variable and function names
- Write clear comments for exported functions
- Follow the existing code structure and patterns

## What to Contribute

- Bug fixes
- New features
- Documentation improvements
- Performance optimizations
- Additional LLM integrations
- UI improvements

## Reporting Issues

Please use the GitHub issue tracker to report bugs or suggest features. Include:
- A clear description of the issue
- Steps to reproduce
- Expected behavior
- Actual behavior
- Your environment (OS, Go version, etc.)

## Pull Request Guidelines

- Keep pull requests focused on a single feature or bug fix
- Write clear commit messages
- Update documentation as needed
- Ensure all tests pass
- Follow the existing code style

## Questions?

If you have questions, feel free to open an issue or contact the maintainers.
